# VM2048

Классическая головоломка 2048, реализованная на языке Jack. Игровое поле 4x4, управление стрелками, цель — объединять одинаковые плитки для получения наибольшего значения.

## Репозиторий
[Исходный код](https://github.com/BceMFredner/VM2048)

## Структура проекта

### Main.jack
Точка входа в приложение. Создает экземпляр игры, запускает основной цикл и корректно освобождает память после завершения.

### Game.jack
Управляет состояниями игры и игровым циклом. Реализует три состояния: главное меню, игровой процесс и экран завершения. Отвечает за обработку ввода с клавиатуры, переключение между экранами, отрисовку счета и максимального значения плитки.

### Grid.jack
Ядро игровой логики. Хранит состояние игрового поля (массив из 16 ячеек), счет и максимальную плитку. Реализует все механики игры: движение плиток, их слияние, генерацию новых значений, проверку окончания игры.

### Board.jack
Отвечает за визуальное представление игрового поля. Рисует рамку, сетку 4x4 и отдельные ячейки. Вычисляет позиции и размеры элементов, определяет какой паттерн использовать для каждого значения плитки.

### Pattern.jack
Содержит 12 уникальных визуальных паттернов (штриховка, сетка, рамки) для отображения различных степеней двойки. Каждый паттерн рисуется вручную через установку пикселей, создавая визуальное различие между плитками без использования цветов.

## Интересные решения

### Алгоритм движения плиток
Вместо написания четырех отдельных методов для каждого направления, реализован один базовый алгоритм [`processLeft`](src/Grid.jack). Остальные направления получаются через комбинацию транспонирования и реверса:
- **Вправо**: реверс строк → влево → реверс строк
- **Вверх**: транспонирование → влево → транспонирование  
- **Вниз**: транспонирование → реверс → влево → реверс → транспонирование

[moveRight](src/Grid.jack), [moveUp](src/Grid.jack), [moveDown](src/Grid.jack)

### Генератор псевдослучайных чисел
Линейный конгруэнтный генератор (LCG) для выбора позиции появления новых плиток. Простая формула `(seed * 17) + 11` с обработкой переполнения обеспечивает достаточную случайность для игры.

[nextRandom](src/Grid.jack)

### Проверка окончания игры
Эффективный алгоритм проверяет не только заполненность поля, но и возможность слияния соседних плиток по горизонтали и вертикали. Игра заканчивается только когда нет ни пустых ячеек, ни возможных ходов.

[checkGameOver](src/Grid.jack)

### Визуальные паттерны
12 уникальных паттернов для плиток от 2 до 4096. Каждый паттерн — комбинация горизонтальных и вертикальных линий с разным шагом, создающая узнаваемый визуальный стиль без использования цветовой палитры.

[Pattern.draw](src/Pattern.jack)

### Конечный автомат
Игровой цикл реализован как конечный автомат с тремя состояниями (0=меню, 1=игра, 2=конец). Переключение между состояниями происходит без рекурсии и дополнительных вызовов функций, что обеспечивает стабильную работу в ограниченной среде.

[Game.run](src/Game.jack)

## Управление
- `ENTER` — запуск новой игры
- `↑ ↓ ← →` — сдвиг плиток
- `SPACE` — перезапуск после проигрыша

## Сборка и запуск
Откройте папку `vm` в VMEmulator.
